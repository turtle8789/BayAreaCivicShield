ARCHITECTURE:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Document Upload Flow:
  User Upload/Camera Input
         ↓
  Streamlit file_uploader or camera_input
         ↓
  Pillow (Image.open, convert format)
         ↓
  PDF2Image (if PDF: convert to images)
         ↓
  Pytesseract (OCR: extract text)
         ↓
  Text Analysis (extract dates, deadlines, etc.)
         ↓
  Deep-Translator (translate to user's language)
         ↓
  Simplification (convert legal jargon)
         ↓
  gTTS (generate audio)
         ↓
  Streamlit Display (show results)
         ↓
  Download Report (TXT or HTML)

HELPER FUNCTIONS:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. extract_dates_from_text(text)
   - Input: Text string
   - Output: List of dates found
   - Uses: 4 regex patterns
   - Formats: MM/DD/YYYY, YYYY-MM-DD, Month DD, YYYY
   - Deduplicates results

2. extract_deadlines_from_text(text)
   - Input: Text string
   - Output: List of deadline sentences
   - Uses: 13 keyword patterns
   - Searches: Full sentences containing keywords
   - Returns: Relevant text excerpts

3. extract_government_agencies_from_text(text)
   - Input: Text string
   - Output: List of agencies found
   - Uses: 7 regex patterns
   - Recognizes: CA agencies, courts, federal agencies
   - Examples: "California Department of", "Superior Court"

4. extract_required_actions_from_text(text)
   - Input: Text string
   - Output: List of action sentences
   - Uses: 16 action keywords
   - Searches: Sentences with must/shall/required
   - Returns: Clear action items

5. simplify_legal_text(text)
   - Input: Legal text
   - Output: Simplified version
   - Uses: 20+ term replacements
   - Converts jargon to plain language
   - Customizable rules

6. page_document_assistant()
   - Main page function (~500 lines)
   - Handles file upload
   - Manages camera capture
   - Orchestrates OCR pipeline
   - Coordinates translation
   - Generates reports
   - Provides UI/UX

PLATFORM-SPECIFIC SETUP:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

WINDOWS:
────────
1. Download Tesseract:
   https://github.com/UB-Mannheim/tesseract/wiki
   Download: tesseract-ocr-w64-setup-v5.x.exe
   
2. Install Tesseract
   Run the installer
   Accept default location
   Complete installation

3. Download Poppler:
   https://blog.alivate.com.au/poppler-windows/
   Download Windows version
   
4. Extract Poppler
   Extract to: C:\Program Files\poppler
   (or add to PATH environment variable)

5. Verify installation:
   python -c "import pytesseract; print('✓ Working')"

6. If error, check:
   - Tesseract installed in default location
   - Poppler in PATH
   - Python restarted after installation

MACOS:
──────
1. Open Terminal

2. Install via Homebrew:
   brew install tesseract poppler

3. Install Python packages:
   pip install pytesseract pdf2image

4. Verify:
   python -c "import pytesseract; print('✓ Working')"

LINUX (Ubuntu/Debian):
──────────────────────
1. Open Terminal

2. Install system libraries:
   sudo apt-get install tesseract-ocr poppler-utils

3. Install Python packages:
   pip install pytesseract pdf2image

4. Verify:
   python -c "import pytesseract; print('✓ Working')"

TROUBLESHOOTING INSTALLATION:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Error: "TesseractNotFoundError"
→ Tesseract not installed, see platform instructions above

Error: "ModuleNotFoundError: No module named 'pytesseract'"
→ Run: pip install -r requirements.txt

Error: "Poppler not found"
→ Poppler utilities not installed, see platform instructions

Error: "Cannot identify image file"
→ Image file corrupted, try different file

Error: Low OCR accuracy
→ Try better quality image (clear, well-lit, high resolution)

See LEGAL_DOCUMENT_ASSISTANT.md > Troubleshooting for detailed guide
